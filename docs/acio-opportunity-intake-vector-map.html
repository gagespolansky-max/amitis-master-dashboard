ACIO Opportunity Intake Vector Map

What gets tracked in Supabase when a Gmail thread is added through the ACIO-Opportunities label and lands in the ACIO deal pipeline. Live schema checked on 2026-05-18. Intake flow updated to use multi-agent analysis.

Executive Map

Primary intake path

Gmail label label:ACIO-Opportunities is scanned by /acio/deals/api/scan. Labeled threads are assumed to be deals and sent through the agent intake workflow.

Core record

Tracked New opportunities create one acio_deals row plus one linked acio_deal_emails row. Later views/enrichment can cache full messages, attachments, and memos.

Intake vector to modify

Agent synthesis + schema The extracted vector is synthesized from email, attachment, and web verification reports, validated by DealExtractionSchema, then inserted into Supabase.

Label Intake Flow

1. User labels thread

Thread is placed under Gmail label ACIO-Opportunities.

2. Scan route runs

runLabelScan searches Gmail for label:ACIO-Opportunities and skips thread IDs already linked to deals.

3. Full thread fetched

Subject, participants, snippets, full message bodies, and attachment metadata are pulled through the Gmail helper.

4. Agents analyze evidence

Email-body, attachment, and low-weight web verification agents each return structured reports.

5. Synthesis + write

Lead intake synthesis creates final fields, then inserts or merges into acio_deals and logs acio_scan_log.

Agent Intake Workflow

Agent Evidence it sees Weight in final synthesis Limitations
email_thread Full Gmail message bodies, sender, date, subject, snippets, and attachment filenames. Highest Used for transaction status, stage, relationship context, and actual conversation facts. Email bodies are capped per message to control token use.
attachments PDF attachments up to the configured byte cap plus text-like attachments such as txt, md, csv, json, html. High Used for strategy, terms, AUM, team, fund/company details, and materials-backed facts. Office files such as docx, xlsx, pptx are currently metadata-only until a conversion path is added.
web_verification Claude web search using the thread metadata and email report. Lowest Used for public context, source checks, and red flags. Absence of public data is not negative evidence. Many ACIO deals are off-market, so web results cannot override email or attachment evidence unless directly contradictory.
lead_synthesis All sub-agent reports. Final Resolves conflicts conservatively and emits the validated Supabase vector. If an optional agent fails, synthesis still uses the reports that succeeded; if the full workflow fails, scan falls back to old metadata extraction.

Current Intake Vector

Vector field Allowed shape Where it lands How it is used
company_name Required string acio_deals.company_name Primary title, dedup target, board card identity.
deal_type fund_allocation, co_invest, direct, or null acio_deals.deal_type Pipeline categorization. If changed to fund_allocation, UI clears vehicle and company_stage.
vehicle spv, direct_equity, safe_convertible, or null acio_deals.vehicle Only meaningful for co-invest or direct opportunities.
company_stage pre_seed, seed, series_a, series_b, series_c_plus, or null acio_deals.company_stage Venture-style stage for direct/co-invest opportunities.
suggested_stage sourced, initial_call, dd_in_progress, ic_review, committed, passed acio_deals.stage Initial board column. Dedup merge can auto-advance a deal, but does not auto-advance into passed.
key_contacts Array of {name,email,role} acio_deals.key_contacts Displayed as people involved. Merge path deduplicates contacts by email.
industry Free text sector acio_deals.industry Shown on cards/panel; editable in UI from a fixed industry list.
company_description 1-2 sentence string or null acio_deals.company_description Deal summary. Enrichment can later rewrite it using full email bodies.
value_proposition 1 sentence string or null acio_deals.value_proposition Amitis-specific fit statement. Enrichment can later rewrite it.

Supabase Tables And Fields

Directly written when a labeled opportunity creates a new deal

Table Fields written by label scan Defaults / constraints
acio_deals company_name, deal_type, stage, status, source, source_thread_id, source_subject, key_contacts, industry, vehicle, company_stage, company_description, value_proposition, created_by_user_id status='confirmed', source='label'. Defaults include priority='medium', timestamps, and generated id.
acio_deal_emails deal_id, thread_id, subject, snippet, last_message_date, participants Links the Gmail thread to the pipeline deal. id and created_at are defaulted.
acio_scan_log scan_type, threads_scanned, new_deals_found, query_used For label mode, scan_type='label' and query_used='label:ACIO-Opportunities [sha:...]'.

Tracked later when the user opens or enriches a deal

Table / storage Fields Triggering action
acio_email_messages deal_email_id, message_id, from_name, from_email, date, subject, body_text, snippet Opening email messages or running enrichment. Full Gmail bodies are cached here.
acio_email_attachments deal_id, deal_email_id, gmail_message_id, gmail_attachment_id, filename, mime_type, size Opening a thread's messages. Stores metadata, not file bytes.
acio_deals notes, priority, reminder_date, reminder_note, memo_url, memo_filename, stage_updated_at, updated_at Manual edits, stage movement, reminders, and memo uploads.
acio-memos storage bucket Stored at {deal_id}/{file.name}; public URL mirrored on acio_deals.memo_url. Uploading an investment memo from the deal panel.
acio_deal_memos deal_id, version, filename, storage_path, notes, created_at Live table exists, but current upload route only updates acio_deals; it does not write version history yet.

Live Schema Notes

Live Supabase columns match the ACIO deal vector.

The live acio_deals table has deal_type, vehicle, company_stage, company_description, and value_proposition. It does not have investment_type.

Code references tables that are absent live.

acio_deal_links and deal_notes do not exist in live Supabase as of this check, even though routes/docs reference them. Link extraction in emails/messages and the links API will fail until that table exists or those routes are revised.

Generated types are missing one live table.

src/lib/database.types.ts includes the core ACIO tables, but not the live acio_deal_memos table.

Security advisory from Supabase MCP.

Supabase reported RLS disabled on several tables, including acio_deal_memos. Do not enable RLS blindly without adding access policies, because that can block application access.

Current Label Prompt Skeleton

Lead intake synthesis:
Use sub-agent reports to produce the final deal fields.

Evidence weight:
1. Email thread bodies: highest.
2. Attachments: high.
3. Web verification: lowest because many deals are off-market.
